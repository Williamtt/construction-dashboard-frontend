# Construction Dashboard 使用者操作手冊

本文件說明系統中**三種身分**各自能做的事、畫面從哪裡進、以及**權限與限制**。文字盡量簡短，方便給實際操作的人閱讀。

---

## 一、先搞懂三個身分

| 身分（系統角色） | 白話說明 | 登入後通常先到哪裡 |
|------------------|----------|-------------------|
| **平台管理員** | 系統營運方，可管理**所有租戶**與跨租戶資料 | **多租後台**（租戶／專案／使用者總覽） |
| **租戶管理員** | 一家公司（一個租戶）的總管，只管**自己公司** | **專案列表**；可再進**單租後台** |
| **專案使用者** | 一般同仁，只在**被指派的專案**裡工作 | **專案列表**；選專案後進**專案工作區** |

**租戶**：在系統裡代表「一間公司／一個組織」。不同租戶的資料互相隔離，彼此看不到。

**專案**：一個工程案。所有儀表板、檔案、日誌、報修等，都是在「某一個專案」底下操作。

---

## 二、整體功能架構（三層）

### 1. 多租後台（僅平台管理員）

路徑前綴：`/platform-admin`

| 功能區 | 主要用途 |
|--------|----------|
| 租戶管理 | 新增／編輯租戶、檢視租戶詳情、**模組開通**（決定該租戶能用哪些功能） |
| 專案總覽 | 跨租戶檢視專案 |
| 使用者總覽 | 跨租戶檢視帳號與角色 |
| 監控與稽核 | 登入紀錄、稽核紀錄等（依實際選單為準） |
| 用量／公告／設定／系統 | 營運相關設定與資訊（依實際選單為準） |

平台管理員可從「返回主應用」回到一般畫面，再進入各專案工作區。

### 2. 單租後台（僅租戶管理員）

路徑前綴：`/admin`

| 功能區 | 主要用途 |
|--------|----------|
| 租戶資訊 | 檢視本公司**模組開通狀態**（多為唯讀；實際開通由平台方設定） |
| 專案管理 | 建立／維護**本租戶**的專案、專案成員 |
| 成員管理 | 管理本租戶帳號、邀請／停用、**權限範本**（新進專案時預設的模組權限） |
| 表單樣板／自主檢查樣板 | 租戶層共用的樣板設定（依權限顯示） |
| 公司設定 | 公司名稱、Logo 等 |

### 3. 專案工作區（三種身分在授權下皆可進入）

路徑形式：`/p/專案ID/...`（網址會帶專案編號，可書籤）

**概況**

- 儀表板、大事記、里程碑  
- 專案資料、契約管理、專案成員、工期調整  

**檔案與表單**

- 檔案管理、相關表單  
- 照片管理（與一般檔案分開授權；照片上傳走照片模組）  

**專案管理（工作執行內）**

- 總覽、WBS、甘特圖、資源庫、排班表、風險與議題  

**施工管理**

- 監測數據、數據上傳、設備管理、自主檢查、進度管理、施工日誌、估驗計價、PCCES 工項、缺失改善、圖說管理、報表、照片管理  

**報修管理**

- 報修總覽、報修紀錄（新增／查閱／處理依權限）  

**手機／PWA**

- 若使用手機瀏覽且未強制使用桌面版，可能導向簡化動線（現場查驗等）；畫面以實際裝置為準。

## 三、權限架構（很重要，看一次就懂八成）

權限分兩條線：**誰能進後台** 與 **在專案裡能做什麼**。

### 1. 系統角色（帳號層級）

| 能力 | 平台管理員 | 租戶管理員 | 專案使用者 |
|------|------------|------------|------------|
| 多租後台 | 可 | 不可 | 不可 |
| 單租後台 | 進 `/admin` 會被導向多租後台 | 可 | 不可 |
| 專案列表範圍 | 全部專案 | 本租戶專案 | 被指派的專案 |
| 專案工作區 | 可（須為專案成員或有准入） | 可 | 可（僅被指派的專案） |

租戶管理員在頂欄或側欄可看到**後台**類入口；專案使用者通常沒有。

### 2. 租戶「模組開通」（功能大開關）

- 平台方在租戶資料中設定**哪些功能模組對該租戶開放**。  
- **被關閉的模組**：該租戶底下**所有人**（含租戶管理員）在專案里都無法使用該模組，畫面上也不會出現對應選單。  
- **平台管理員**不受租戶模組開通限制（方便維運）。  
- 若租戶尚未完成模組開通設定、或等效上「等於全部關閉」，租戶管理員可能**無法新增專案**、也**無法調整**成員權限範本與專案內覆寫（與後端規則一致；實際以畫面提示為準）。

### 3. 專案模組權限（細部：讀／新增／修改／刪除）

在「模組已開通」的前提下，每位專案成員還有四種動作（可分開勾選）：

| 動作 | 一般用途 |
|------|----------|
| **讀取** | 看列表、看內容、進入該功能頁 |
| **新增** | 新增一筆、上傳、建立 |
| **修改** | 編輯內容、變更狀態（不含刪除） |
| **刪除** | 刪除、移除、作廢 |

**誰決定預設？**

- **租戶後台 → 成員管理**：可設定「權限範本」——使用者**第一次被加進專案**時，會複製這份預設。  
- **專案內 → 專案成員**：租戶管理員或平台管理員可對**單一成員**做**覆寫**（不影響其他人）。  

**誰不受這四格細項限制？**

- **平台管理員**、**租戶管理員**在專案內會視為具備完整操作空間（與系統設計一致）。  
- **專案使用者**則完全依上列四種權限。

### 4. 專案成員與「成員權限頁」

- 一般專案功能的成員名單、加入／移除專案，仍屬**專案管理**流程。  
- 「每位成員的模組矩陣覆寫」僅**租戶管理員**與**平台管理員**可調整。

### 5. 檔案與照片（容易混淆）

- **一般檔案、表單**：多半對應「數據上傳、檔案／表單」類模組。  
- **圖說**：對應「圖說管理」模組。  
- **照片圖庫**：對應「照片管理」模組；上傳照片時系統會用照片類別，**與一般檔案分開檢查權限**。  
- 只開照片、未開一般檔案時，仍可能在照片功能裡上傳（依系統設計）。

## 四、平台管理員 — 操作說明

### 你能做什麼

- 管理所有租戶、專案、使用者。  
- 設定租戶的**模組開通**（決定客戶能用哪些專案內功能）。  
- 進入任何有權的專案工作區；不受租戶模組關閉影響。

### 建議操作步驟（新客戶開通）

1. 登入系統 → 進入**多租後台**。  
2. **租戶管理**：建立租戶或開啟既有租戶。  
3. 進入租戶詳情／管理頁：設定**模組開通**（勾選要開放的功能）。  
4. 建立或指定至少一位 **租戶管理員** 帳號（在使用者／租戶流程中完成，依實際畫面）。  
5. 請客戶用租戶管理員登入，在**單租後台**建立專案並加入成員。

### 規則與限制（摘要）

- 專案使用者**不能**進多租後台。  
- 模組開通是租戶級大開關；未開的模組，客戶全員都看不到。

## 五、租戶管理員 — 操作說明

### 你能做什麼

- 管理**自己公司**的專案與帳號。  
- 設定成員的**權限範本**（新進專案預設權限）。  
- 在專案里覆寫**單一成員**的模組權限。  
- 使用專案內功能時，與專案使用者相同入口，但不受「四格細項」限制。

### 建議操作步驟（日常）

1. 登入 → **專案列表** → 點專案進入工作區，或點**後台**進 **單租後台**。  
2. **專案管理**：新增專案、編輯專案、將同事加為專案成員。  
3. **成員管理**：新增帳號或邀請、停用離職同仁、區分內部／外部成員（若公司有使用）。  
4. 在成員管理中設定**權限範本**，讓之後加入專案的人帶入合理預設。  
5. 若某人需要特例：進**專案** → **專案成員** → 進入該成員的**權限覆寫**畫面調整。

### 規則與限制（摘要）

- **看不到**其他租戶的資料。  
- 平台方**關閉**的模組，你無法在專案內開給任何人。  
- 若模組開通未就緒或全部被關閉，可能無法**新增專案**或無法編輯權限矩陣（依畫面為準）。  
- 嘗試開啟無權網址時，系統會擋下或導向安全頁面。

## 六、專案使用者（一般同仁）— 操作說明

### 你能做什麼

- 在**專案列表**中，只看得到**被指派**的專案。  
- 進入專案後，側欄只會出現你有**讀取**權限的功能。  
- 按鈕（新增、編輯、刪除）會依你的**四種權限**顯示或隱藏。

### 建議操作步驟

1. 登入 → **專案列表** → 點選專案。  
2. 用左側選單切換：概況、監測、契約、檔案、工作執行（專案管理／施工管理／報修）等。  
3. 若某選單不見：代表沒有讀取權限，或租戶未開通該模組 → 請洽**租戶管理員**。  
4. 若看得到列表但無法按「新增」或「刪除」：代表僅有讀取或部分權限 → 同樣洽管理員調整**專案成員權限**。  
5. 使用搜尋、篩選、分頁、匯出（若有）時，僅會看到自己權限內的資料。

### 規則與限制（摘要）

- **沒有**單租／多租後台入口（一般情況）。  
- 無法把別人加進專案或改公司設定。  
- 登入逾時或帳號停用時，請重新登入或聯絡管理員。

## 八、常見狀況排除

| 狀況 | 可能原因 | 建議 |
|------|----------|------|
| 側欄少很多功能 | 租戶未開通模組，或個人沒有讀取權 | 專案使用者找租戶管理員；管理員找平台方檢查模組開通 |
| 看得到但無法按「新增／刪除」 | 僅有讀取或部分動作權限 | 租戶管理員調整專案成員權限覆寫 |
| 無法新增專案 | 租戶模組開通未設定或全部被關閉 | 平台管理員調整租戶模組開通 |
| 登入後馬上被踢回登入 | Token 過期或帳號停用 | 再登入一次或聯絡管理員 |
| 平台管理員想從後台回專案 | 使用「返回主應用」再到專案列表 | — |
